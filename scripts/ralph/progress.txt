# Ralph Progress Log
Started: Wed Mar  4 21:45:19 UTC 2026

## Codebase Patterns
- Settings stored as key-value pairs in `settings` table; JSON arrays stored as stringified JSON
- Use `INSERT OR IGNORE` for seeding defaults (idempotent, won't overwrite user changes)
- Frontend `updateSettings` takes an object with optional fields, not positional args
- Valid provider names come from `getAllProviders()` in the registry; valid skills are `prd`, `prd-questions`, `ralph`
- Backend routes import from providers with `.js` extension (ESM)
